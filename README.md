# ProjectPy AI

ProjectPy AI 是一个“从项目开始学 Python”的 AI 学习网站 MVP。它不把用户先丢进语法目录，而是让用户从真实小项目开始，在写代码、运行代码、看报错和 AI 提示的过程中逐步理解 Python。

## 项目亮点

- 项目式学习：内置待办事项工具、文件夹整理器、天气查询工具、数据分析助手、网页标题提取器、猜数字小游戏。
- 任务拆解：每个项目被拆成可完成的小任务，任务会标记需要用到的知识点。
- 在线运行：在网页里编辑 Python 代码，点击运行后直接看到输出或错误。
- AI 导师：不填 API Key 时使用本地导师提示；填写自己的 OpenAI API Key 后使用 OpenAI 生成个性化反馈。
- 学习进度：项目、任务、代码、导师对话和完成状态会保存在浏览器本地。
- 响应式界面：支持桌面和手机宽度下使用。

## 重要安全说明

这个项目不会读取开发者自己的 OpenAI API Key，也不会默认使用服务端环境变量里的 Key。

用户如果想使用 OpenAI AI 导师，需要在网页右侧“AI 导师”区域填写自己的 OpenAI API Key。Key 会保存在当前浏览器的 `localStorage`，并只在请求导师反馈时发送给本机运行的 `server.py`。

如果不填写 API Key，网站仍然可以使用本地导师规则完成演示和基础学习。

## 本地运行

要求：

- Python 3.10 或更新版本

启动：

```bash
python3 server.py
```

打开浏览器访问：

```text
http://127.0.0.1:8000
```

## 使用自己的 OpenAI API

1. 启动项目并打开网页。
2. 在右侧“AI 导师”区域找到 API 设置。
3. 填入自己的 OpenAI API Key。
4. 模型默认是 `gpt-5.4-mini`，也可以手动改成其他可用模型。
5. 点击“保存”。
6. 再点击“提示”“报错”“总结”或直接向 AI 提问。

清除 API Key：

1. 点击“清除”。
2. 页面会回到本地导师模式，不再调用 OpenAI。

## 功能说明

### 项目库

左侧展示不同项目方向，每个项目包含难度、预计时间、项目成果和学习目标。

### 任务拆解

每个项目包含多个任务。用户完成任务后，系统会根据代码内容和运行输出做基础验收，并更新进度。

### 代码编辑和运行

中间区域提供 Python 代码编辑器和运行按钮。运行结果会显示在“运行输出”，报错会显示在“错误信息”。

### AI 导师

AI 导师会结合当前项目、当前任务、用户代码、运行输出和错误信息给反馈。默认策略是先给提示和思路，不直接给完整答案。

## API 设计

### POST `/api/run`

运行用户代码。

请求：

```json
{
  "code": "print('hello')"
}
```

返回：

```json
{
  "ok": true,
  "output": "hello\n",
  "error": "",
  "duration_ms": 32
}
```

### POST `/api/mentor`

生成导师反馈。不传 `apiKey` 时使用本地导师；传入用户自己的 `apiKey` 时调用 OpenAI。

请求字段包括：

- `message`
- `code`
- `output`
- `error`
- `projectTitle`
- `taskTitle`
- `taskGoal`
- `concepts`
- `apiKey`
- `model`
